> **所属项目**：165/50 零基础增肌项目（`~/projects/fitness-muscle-building/`）

> **启动日期**：2026-03-09
> **状态**：v2（审查修订版）

## 一、定位（为什么做这个）

### 问题陈述

V2 方案（增肌计划-V2.md）定义了完整的追踪机制，但执行层面只有：
- 「薄荷健康」App 记录热量（第三方，不可定制）
- 手机备忘录记录训练重量（非结构化，难以回顾趋势）
- 每周五 5 分钟复盘（无工具支撑，容易跳过）

**核心矛盾**：V2 的追踪标准是**多维**的（体重 + 围度 + 力量 + 照片 + 周复盘），但现有工具是**单维**的。

### 一句话定义

一个**零后端、手机可用的单页 Web 应用**，将 V2 方案的核心追踪需求（体重/训练重量/饮食达标/周复盘）整合到一个界面中。

### MVP 范围定义（v1→v2 变更，来源：🔴1）

> **Phase 5 只交付 MVP**——用户每天能用、数据不丢、能看到趋势。
> 高级功能（问题诊断决策树、里程碑自动判定、围度追踪）放入 Phase 5.5。

| MVP（Phase 5 交付）| 后续迭代（Phase 5.5）|
|---|---|
| ✅ 仪表盘 | ⏳ 问题诊断交互决策树 |
| ✅ 体重追踪 + 趋势图 | ⏳ 围度追踪（腰/胸/臂） |
| ✅ 训练记录（A/B/C 模板）| ⏳ 里程碑自动判定 |
| ✅ 饮食简要记录 | ⏳ 训练重量进步曲线 |
| ✅ 周复盘 | ⏳ PWA / Service Worker |
| ✅ 数据导出/导入 | |

### Phase 间价值链

| 前序 Phase | 产出 | 本 Phase 如何使用 |
|---|---|---|
| Phase 3（V2 方案）| 追踪指标定义 | 作为功能需求来源 |

## 三、核心用户场景（v1→v2 新增，来源：🔴3）

> 每个场景描述用户的实际操作流，是验收标准的核心依据。

### 场景 1：训练日记录（每周 3 次，使用频率最高）

```
打开 app（手机浏览器）
  → 首页仪表盘：看到「本周已训练 1/3」
  → 点「记录训练」
  → 选训练类型 A
  → 看到预填的 6 个动作（杠铃深蹲/卧推/...）
  → 逐动作输入重量和完成次数
  → 点「完成训练」
  → 仪表盘更新：「本周已训练 2/3」
全程 < 3 分钟
```

### 场景 2：每日体重记录（每天早起，30 秒完成）

```
打开 app
  → 仪表盘显示「今日体重未记录」提示
  → 点「记录体重」
  → 输入体重 51.2
  → 仪表盘更新：体重趋势图多了一个点
  → 看到「距离目标 62kg 还差 10.8kg」
全程 < 30 秒
```

### 场景 3：周五复盘（每周 1 次，5 分钟）

```
打开 app（周五晚上）
  → 仪表盘提示「本周还没有复盘」
  → 点「周复盘」
  → 看到自动预填数据：本周训练 3 次 ✅ / 蛋白质达标 4/7 天 / 体重 +0.3kg
  → 可以修改或添加备注
  → 点「提交复盘」
  → 仪表盘显示「连续复盘 4 周 🔥」
```

### T2: UI 设计系统 + 路由

**技术方案**：
- Google Fonts: Inter（唯一外部 CDN 依赖）
- CSS 变量深色主题
- 移动优先（375px）
- emoji 代替 SVG 图标（v1→v2 变更，来源：🟢3）

**路由方案**（v1→v2 明确，来源：🟡3）：
- 使用 hash 路由：`#/dashboard`, `#/weight`, `#/training`, `#/more`
- `window.addEventListener('hashchange', ...)` 监听路由变化
- 每个「页面」是一个 `<section class="page">` 元素
- 切换路由时：隐藏所有 section → 显示目标 section

**底部导航**（v1→v2 变更，来源：🟢2）：
- 4 个 Tab：📊 仪表盘 / ⚖️ 体重 / 🏋️ 训练 / ⚙️ 更多
- 「更多」包含：饮食记录 / 周复盘 / 数据导出导入 / 问题诊断（Phase 5.5 链接）

**视觉方向**：
- 深色主题：背景 #0a0a1a，卡片 #1a1a2e
- 渐变强调色：蓝→紫 (#4f7df7 → #7c3aed)
- 圆角卡片 16px，微阴影
- 数据数字用大号字体（32-48px）突出
- 状态色：成功 #10b981 / 警告 #f59e0b / 危险 #ef4444

### T4: 体重追踪模块

**功能**：
- 输入框：日期（默认今天）+ 体重（kg）
- 记录列表：最近 N 条记录
- 趋势可视化：**CSS 柱状图**（v1→v2 变更，来源：🔴2）
  - 用 CSS Grid + 高度百分比实现柱状图
  - 柱子高度 = (当前体重 - 最小值) / (最大值 - 最小值) × 100%
  - 三个视图切换：7 天 / 30 天 / 全部
- 统计卡片：本周 vs 上周体重变化

### T5: 训练记录模块

**功能**：
- 选择训练类型（A / B / C）→ 自动加载动作列表
- 每个动作显示：名称 + 目标肌群
- 每个动作下方：3 组输入行（重量 kg + 完成次数）
- 「上次记录」提示：「上次：40kg × 10」（来自历史数据）
- 渐进超负荷提示：如果有上次记录，建议「挑战 +2.5kg？」
- 保存后跳回仪表盘

### T6: 饮食 + 周复盘（合并模块）

**饮食记录**：
- 快速输入：今日热量（大卡）+ 蛋白质（g）
- 达标判定：≥ 2300 大卡 + ≥ 90g 蛋白质 → ✅
- 本周达标天数统计

**周复盘**：
- 三个核心检查项（V2 来源）自动预填：
  1. 本周训练次数（从 trainingLog 统计）
  2. 蛋白质达标天数（从 nutritionLog 统计）
  3. 体重变化（从 weightLog 计算最早到最晚差值）
- 用户可添加备注
- 复盘历史列表

### T7: 浏览器验收测试

按核心用户场景逐一验证（来源：🔴3 的场景 1/2/3）。
