| 决策 | 为什么这么做 | 效果 |

## 三、新发现的可复用模式

### 模式 1：Dogfooding 验证 — 用 Skill 自身流程优化 Skill

```
场景：优化一个流程/方法论/Skill
做法：用这个 Skill 定义的流程来执行优化任务
价值：在实际执行中发现流程自身的问题（如质量门过严、缺少某些环节）
本次验证：用 full-cycle-builder 的 5 Phase 流程来优化 full-cycle-builder 自身
```

### 模式 2：内部案例回溯 > 外部竞品调研

```
场景：优化一个已经在使用中的工具/流程
发现：自己的两个项目的实际执行偏差分析，比 7 个外部竞品的对比更有价值
原因：外部竞品是别人的环境和约束，内部案例是自己的真实数据
推论：当调研对象是"优化现有方案"时，内部回溯应该优先于外部竞品
```

### 模式 3：互斥边界必须包含"前序→后续关系"

```
场景：多个 Skill 有重叠的 Phase 定义
发现：project-planner 和 full-cycle-builder 的 Phase 有部分重叠
解法：不是互斥关系，是前序→后续关系（0→1 vs 1→100）
推论：互斥边界表不仅要标"不要用这个"，还要标"先用那个再用这个"
```
