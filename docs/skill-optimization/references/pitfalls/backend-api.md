# 后端 API 踩坑预警库

> **适用**：后端 API / 服务端项目
> **维护方式**：后端项目经验沉淀时追加

---

## 🔴 P0: 错误处理不能只返回 500

```
场景：所有异常都返回 HTTP 500 Internal Server Error
根因：catch 块中没有区分错误类型
修复：区分 4xx（客户端错误）和 5xx（服务端错误），返回有意义的错误码和消息
预防：定义统一的错误响应格式
```

## 🟡 P1: 数据库迁移必须可回滚

```
场景：schema 变更后发现问题，需要回退
修复：每个迁移脚本同时准备 up 和 down 操作
预防：部署前在测试环境验证回滚脚本
```

## 🟡 P2: 环境变量不能硬编码

```
场景：数据库密码/API Key 写在代码里
修复：所有敏感信息使用环境变量（.env 文件 + 读取库）
预防：.gitignore 排除 .env，提供 .env.example 模板
```

## 🟡 P3: 并发写入需要考虑竞态条件

```
场景：多个请求同时修改同一资源，数据不一致
修复：使用事务（Transaction）或乐观锁（Version 字段）
预防：设计阶段识别可能的并发写入场景
```
