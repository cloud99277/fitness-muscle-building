# 质量门预设方案

> **用途**：Phase 0 判定项目规模后，选择对应的质量门配置

---

## Small 项目（~1天）

适用：小工具、脚本、单文件修改、快速原型

| 质量门 | 阈值 | 说明 |
|--------|------|------|
| A（调研）| **跳过** | 无需正式调研，直接规划 |
| B（规划）| 7.5/10 | 有基本规划即可 |
| C（编码）| 运行时零错误 | 无需正式 code-review |
| D（部署）| **可选** | 视项目需要 |

---

## Medium 项目（~3天）

适用：功能模块、小型应用、Skill 开发

| 质量门 | 阈值 | 说明 |
|--------|------|------|
| A（调研）| 8.0/10 | Quick Mode 调研即可 |
| B（规划）| 8.0/10 | 需要完整规划文档 |
| C（编码）| 零错误 + Code Review | code-review 至少跑 Standard 模式 |
| D（部署）| 目标环境验收通过 | 需要在目标环境测试 |

---

## Large 项目（1周+）

适用：完整应用、多模块系统、跨阶段项目

| 质量门 | 阈值 | 说明 |
|--------|------|------|
| A（调研）| 8.5/10 | Full Mode 调研 + 反面教材分析 |
| B（规划）| 8.0/10 | 需要完整规划 + 核心用户场景 |
| C（编码）| 零错误 + CR + 安全检查 | code-review + 安全层检查 |
| D（部署）| 目标环境全功能验收 | 多环境/多设备测试 |

---

## 自定义

用户可以在 Phase 0 中调整质量门阈值。以上预设仅为推荐值。
