## 🎯 Sprint 5 — 今日计划功能

> **目标**：App 从"事后记录器"升级为"每日执行指引"，用户打开就能看到今天该做什么。

### 需求描述

当前 App 的问题：用户知道要训练、要吃够、要早睡，但打开 App 不知道"今天具体干什么"。
需要在仪表盘最显眼位置新增「今日计划」卡片，自动根据星期几生成当天任务清单。

### 任务拆解

| ID | 任务 | 说明 | 状态 |
|---|---|---|---|
| A11 | 今日计划卡片 | 仪表盘顶部插入「今日计划」卡片组件 | 待开发 |
| A12 | 训练日逻辑 | 根据星期自动判断：周一=推A、周三=拉B、周五=腿C、其余=休息 | 待开发 |
| A13 | 动作清单展示 | 训练日显示具体动作+组×次，点击可跳转训练记录页 | 待开发 |
| A14 | 每日必做清单 | 饮水/凯格尔/23:30睡 三项可勾选（今日内有效） | 待开发 |
| A15 | 降级模式切换 | 高压力日可切换降级模式，自动缩减动作列表（2-3个主动作） | 待开发 |
| A16 | 营养目标提示 | 显示今日营养目标（基础期 2000kcal+75g蛋白 / 正常期 2300kcal+100g蛋白） | 待开发 |

### 设计规格

#### 卡片位置
- 仪表盘最顶部，在「今日体重未记录」提示下方
- 紧接在进度环卡片上方

#### 训练日卡片内容

━━━━━━━━━━━━━━━━━━━━

  2. 俯卧撑          3组 × 力竭       [记录→]

  ☐ 23:30 前睡

#### 降级模式
- 勾选"高压力日"后，动作列表缩减为：
  - 推A → 地板卧推 + 俯卧撑 + 肩推（3个主动作，每组2组）
  - 拉B → 俯身划船 + 单臂划船 + 弯举（3个主动作，每组2组）
  - 腿C → 高脚杯深蹲 + 箭步蹲 + 平板支撑（3个主动作，每组2组）
- 预计时长改为 20-30 分钟
- 训练记录保存时标记 `mode: 'downgrade'`（已有逻辑）

#### 数据源
- 训练模板：复用 `Store.getTemplates()` 中的 A/B/C 模板
- 星期判断：`new Date().getDay()`（0=周日, 1=周一, ..., 6=周六）
- 营养目标：复用 `V41_TARGETS` 常量
- 必做清单状态：存储在 `localStorage` 中，key 为 `daily-checklist-{date}`，每日自动重置

#### 交互
- 点击动作名 → 跳转到对应训练记录页（`App.startTraining(type)`）
- 勾选必做清单 → 本地存储，刷新不丢失
- 切换降级模式 → 动态替换动作列表，无需刷新页面

### 技术要点

1. 新增 `Components.renderTodayPlan()` 方法
2. 在 `renderDashboard()` 顶部调用
3. 新增 `Store.getDailyChecklist(date)` / `Store.saveDailyChecklist(date, data)` 方法
4. 周日判断：周一=1, 周三=3, 周五=5 为训练日，其余为休息日
5. 降级模式用一个 toggle checkbox 控制，通过 CSS class 切换显示

## 💡 Sprint 5+ — Telegram 训练记录集成

> **目标**：通过 Telegram 消息快速记录训练数据，不需要打开 App。

### 需求描述

训练时手边不一定方便打开 App 填表单。希望通过 Telegram 发一条消息就能记录训练。
需要在 Hermes/OpenClaw 中创建一个 skill，解析自然语言训练记录并持久化。

### 方案对比

| 方案 | 机制 | 跟 App 联动 | 开发量 | 推荐场景 |
|---|---|---|---|---|
| A 轻量版 | Telegram → 本地 JSON 文件 | ❌ 独立 | 5 分钟 | 先跑起来，不依赖 App |
| B 完整版 | Telegram → Firebase REST API | ✅ 打通 | 30 分钟 | 数据统一，App 直接可见 |
| C 中间版 | Telegram → 本地 JSON + 定期导入 App | ⚠️ 需手动导入 | 15 分钟 | 不碰 Firebase，数据最终汇入 |

### 用户输入示例

```
练了推A
卧推 6.5kg×12, 6.5kg×10, 6.5kg×10, 6.5kg×8
俯卧撑 15, 12, 10
肩推 6.5kg×12, 6.5kg×10, 6.5kg×10
侧平举 4kg×15, 4kg×13, 4kg×12
臂屈伸 6.5kg×12, 6.5kg×10, 6.5kg×10
钻石俯卧撑 12, 10
```

或更简洁：
```
推A完成 卧推6.5×12×4 俯卧撑15×3 肩推6.5×12×3
```

### Skill 功能要求

1. 解析自然语言训练记录（动作名、重量、次数、组数）
2. 自动识别训练类型（推A/拉B/腿C/GA/GB）
3. 支持降级标记："降级训练" / "很累只做了3个动作"
4. 持久化到结构化文件或 Firebase
5. 回显确认："✅ 推A 已记录：6个动作，22组"
6. 支持补记："补记昨天的训练：..."

### 优先级

- **P2**（不紧急，有了更好）
- 等 Sprint 5 今日计划功能完成后，再考虑集成方式
- 方案 A 可以随时做，不影响 App
